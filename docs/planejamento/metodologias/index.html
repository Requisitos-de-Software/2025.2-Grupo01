O processo de definição e compreensão das necessidades e restrições de um novo sistema é conhecido como Engenharia de Requisitos. Uma de suas fases mais críticas é a elicitação de requisitos, um termo que se consolidou como uma versão do inglês elicit requirements. Segundo Vazquez e Simões (2016, p. 140), a elicitação é “um processo de aquisição de conhecimento, onde se aplicam técnicas para compreender melhor o negócio a ser impactado pelo projeto, para identificar partes interessadas e para identificar e refinar os tipos de requisitos”.

Este processo é fundamental, pois falhas na identificação das reais necessidades dos usuários podem levar a graves consequências no desenvolvimento do projeto. O analista de requisitos não deve assumir uma postura passiva, mas sim atuar de forma proativa para descobrir inclusive os "requisitos implícitos" — desejos que a parte interessada não manifesta por acreditar que são óbvios ou por serem ainda subconscientes.

Para alcançar esse objetivo, o analista dispõe de uma "caixa de ferramentas" com diversas técnicas de elicitação. A escolha da técnica mais adequada depende de fatores como o domínio do negócio, a cultura da empresa e os recursos disponíveis, sendo comum a combinação de múltiplas abordagens ao longo de um projeto.

Técnicas de Elicitação

  • Brainstorming: Técnica de grupo para gerar um grande volume de ideias criativas sobre um tema, de forma colaborativa e sem julgamentos. (Vazquez; Simões, 2016)
  • Entrevista: Diálogo planejado para obter informações detalhadas sobre as necessidades dos interessados. (Vazquez; Simões, 2016)
  • Pesquisa/Questionário: Coleta rápida de informações de um público numeroso ou disperso geograficamente. (Vazquez; Simões, 2016)
  • Análise de Documentos: Estudo da documentação existente para entender o negócio. (Vazquez; Simões, 2016)
  • Observação (Etnografia): Observar o trabalho no ambiente real para descobrir requisitos implícitos. (Vazquez; Simões, 2016)
  • Elaboração de Glossário: Criação de uma lista de termos e definições para unificar o vocabulário do projeto. (Vazquez; Simões, 2016)
  • Controle de Questões: Registrar e acompanhar dúvidas que surgem durante a elicitação. (Vazquez; Simões, 2016)

VAZQUEZ, Carlos Eduardo; SIMÕES, Guilherme Siqueira. Engenharia de Requisitos: Software Orientado ao Negócio. Rio de Janeiro : Brasport, 2016
